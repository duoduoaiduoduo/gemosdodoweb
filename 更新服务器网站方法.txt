### ⚠️ 第零阶段：一次性改名迁移（只需在服务器上做一次）

项目已从 `zip` 更名为 `gemosdodoweb`。本地代码和文档已全部改好，但**服务器上的目录名和 PM2 进程名还是旧的**，需要你登录服务器执行以下命令迁移一次。做完这一次后，以后就按下面的“日常三阶段”正常用即可。

```bash
# 0-1. 变身管理员
sudo su -

# 0-2. 停掉并删除旧的 PM2 进程
pm2 delete zip-site

# 0-3. 把网站目录改名（从 zip 改成 gemosdodoweb）
mv /www/wwwroot/zip /www/wwwroot/gemosdodoweb

# 0-4. 进入新目录，拉取包含改名的最新代码
cd /www/wwwroot/gemosdodoweb
git pull origin master

# 0-5. 用新名字启动服务（deploy.sh 里已改为 gemosdodoweb-site）
pm2 start server.js --name gemosdodoweb-site
pm2 save

# 0-6. 如果你在宝塔/Nginx 里配置过站点根目录指向 /www/wwwroot/zip，
#      记得去面板把网站根目录改成 /www/wwwroot/gemosdodoweb，然后重载 Nginx。
```

做完上面这一次后，以后就完全不用再管改名的事了，直接走下面的日常流程。

为了方便你以后随时查阅，我为你整理了这份**“日常网站更新终极工作流”**。你可以把这段保存到你的电脑备忘录里。

以后你每次想要修改网站内容、增加新功能，只需要无脑执行以下这三个阶段：

---

### 第一阶段：在本地电脑修改并上传（你的 PC）

在你自己的电脑上，用 VS Code 打开代码，像往常一样修改你的网页（比如修改文字、调整颜色、增加功能）。改完并保存后，在 VS Code 的终端里依次执行：

```powershell
# 1. 把所有改动添加到暂存区
git add .

# 2. 给这次更新写个简单的备注（双引号里的字自己随便写）
git commit -m "修复bug"

# 3. 把代码推送到 Gitee 云端仓库
git push
```
*(只要没报错，就说明新代码已经安全存到云端了)*

---

### 第二阶段：在服务器一键自动部署（你的服务器）

打开你的服务器终端工具（比如 FinalShell、Xshell 等），连接上服务器后，依次执行这三行终极指令：

```bash
# 1. 变身最高管理员大老板（如果需要密码，请盲打你的服务器密码后回车）
sudo su -

# 2. 进入你的网站大本营
cd /www/wwwroot/gemosdodoweb

# 3. 启动魔法自动化脚本
./deploy.sh
```

**按下回车后，你可以端起茶杯，看着屏幕上的脚本自动为你表演：**
1. 备份你的 `data.json` 和 `uploads` 图片
2. 从 Gitee 拉取你刚刚提交的最新代码
3. 安装可能新增的依赖包
4. 重新打包前端 React/Vue 页面
5. 指挥 PM2 管家重启 Node.js 服务

---

### 第三阶段：见证奇迹（浏览器）

当服务器终端显示 `== 🎉 完美！全自动更新执行完毕！ ==` 时，去你的浏览器里打开你的网站，按下：

**`Ctrl + F5` （强制刷新，清除浏览器旧缓存）**

你就能瞬间看到你刚刚在本地修改的最新效果了！
